# 数据集路径与模型保存路径修改说明

这份笔记用于梳理当前预训练入口里两条容易混淆的路径链路：

- 数据集路径 `data_path` 是如何从命令行传到数据集加载器中的
- 模型保存路径 `save_dir` 是如何从命令行传到权重保存、续训恢复和权重加载中的

本文只对应当前的 `pretrain` 训练流程，不展开到项目中的其他脚本。

## 1. 数据集路径的传递链路
在 `trainer/train_pretrain.py` 中，预训练数据路径通过命令行参数传入：

```python
parser.add_argument("--data_path", type=str, default="dataset/pretrain_t2t_mini.jsonl")
```

训练入口拿到 `args.data_path` 后，会直接传给 `PretrainDataset(...)`：

```python
train_ds = PretrainDataset(args.data_path, tokenizer, max_length=args.max_seq_len)
```

真正的数据集路径解析发生在 `dataset/lm_dataset.py` 中：

```python
data_path = Path(data_path)
if not data_path.is_absolute():
    data_path = (Path(__file__).resolve().parents[1] / data_path).resolve()
self.samples = load_dataset('json', data_files=data_path.as_posix(), split='train')
```

这里的逻辑是：

- 如果传入的是绝对路径，例如 `E:/Downloads/pretrain_t2t_mini.jsonl`，就直接使用
- 如果传入的是相对路径，例如 `dataset/pretrain_t2t_mini.jsonl`，就会基于仓库根目录补全成绝对路径
- 最后统一交给 `load_dataset('json', ...)` 读取

所以当前代码同时支持两种写法：

```bash
python .\trainer\train_pretrain.py --data_path dataset/pretrain_t2t_mini.jsonl
python .\trainer\train_pretrain.py --data_path E:/Downloads/pretrain_t2t_mini.jsonl
```

## 2. 模型保存路径的传递链路
在 `trainer/train_pretrain.py` 中，模型保存目录通过 `--save_dir` 指定，默认值来自 `DEFAULT_OUT_DIR`：

```python
parser.add_argument("--save_dir", type=str, default=str(DEFAULT_OUT_DIR))
args.save_dir = str(resolve_project_path(args.save_dir, DEFAULT_OUT_DIR))
```

这一步的作用是：

- 默认保存到仓库下的 `out/`
- 如果用户传入相对路径，就基于仓库根目录解析
- 如果用户传入绝对路径，就直接使用

例如：

```bash
python .\trainer\train_pretrain.py --save_dir out
python .\trainer\train_pretrain.py --save_dir E:/Workplace/tmp_out
```

### 2.1 纯权重文件保存
训练循环中，纯模型权重仍然由 `torch.save(...)` 直接写到 `args.save_dir`：

```python
ckp = f'{args.save_dir}/{args.save_weight}_{lm_config.hidden_size}{moe_suffix}.pth'
torch.save({k: v.half().cpu() for k, v in state_dict.items()}, ckp)
```

默认情况下，常见文件名类似：

```text
out/pretrain_768.pth
```

### 2.2 resume checkpoint 保存
除了纯权重外，还会调用 `lm_checkpoint(...)` 保存一份完整的续训 checkpoint：

```python
lm_checkpoint(
    lm_config,
    weight=args.save_weight,
    model=model,
    optimizer=optimizer,
    scaler=scaler,
    epoch=epoch,
    step=step,
    wandb=wandb,
    save_dir=args.save_dir,
)
```

`lm_checkpoint(...)` 内部会生成两类路径：

```python
ckp_path = save_dir / f'{weight}_{lm_config.hidden_size}{moe_path}.pth'
resume_path = save_dir / f'{weight}_{lm_config.hidden_size}{moe_path}_resume.pth'
```

其中和续训直接相关的是：

```text
out/pretrain_768_resume.pth
```

这份文件除了模型参数外，还会保存：

- optimizer 状态
- scaler 状态
- epoch
- step
- world_size
- wandb_id

## 3. 本次路径修改点
这次修改的目标，不是重写训练逻辑，而是把 `train_pretrain.py` 中和输出目录相关的调用统一到 `args.save_dir`。

最终只改了 3 处调用：

### 3.1 保存 checkpoint 时改为写入 `args.save_dir`
原来：

```python
lm_checkpoint(..., save_dir=DEFAULT_CHECKPOINT_DIR)
```

现在：

```python
lm_checkpoint(..., save_dir=args.save_dir)
```

这样 `_resume.pth` 不会再单独写到 `checkpoints/`，而是和纯权重一起落到 `out/`。

### 3.2 续训恢复时改为从 `args.save_dir` 读取
原来：

```python
ckp_data = lm_checkpoint(lm_config, weight=args.save_weight, save_dir=DEFAULT_CHECKPOINT_DIR) if args.from_resume == 1 else None
```

现在：

```python
ckp_data = lm_checkpoint(lm_config, weight=args.save_weight, save_dir=args.save_dir) if args.from_resume == 1 else None
```

这样 `from_resume=1` 会去当前保存目录里找 `_resume.pth`。

### 3.3 加载已有权重时也改为使用 `args.save_dir`
原来：

```python
model, tokenizer = init_model(lm_config, args.from_weight, device=args.device)
```

现在：

```python
model, tokenizer = init_model(lm_config, args.from_weight, save_dir=args.save_dir, device=args.device)
```

这样 `from_weight` 和保存路径保持一致，不会出现“权重保存在 `out/`，但加载时去别的目录找”的问题。

## 4. 为什么没有继续改 `trainer_utils.py` 的默认值
这次修改刻意保持为最小 diff，只让 `train_pretrain.py` 通过调用参数控制输出目录。

因此 `trainer_utils.py` 里的 `lm_checkpoint(...)` 默认行为仍然保留为：

```python
def lm_checkpoint(..., save_dir=DEFAULT_CHECKPOINT_DIR, ...)
```

也就是说：

- 通用工具函数默认仍然认为 checkpoint 在 `checkpoints/`
- 但 `train_pretrain.py` 这个入口脚本已经显式传入 `args.save_dir`
- 所以当前预训练流程的实际输出目录，最终由调用方控制，而不是由工具函数默认值控制

这样做的好处是：

- 改动小
- 不影响其他可能还依赖 `lm_checkpoint()` 默认行为的脚本
- 更符合“只改调用点，不重构工具层”的目标

## 5. 最终效果与使用建议
对于当前 `train_pretrain.py`，默认情况下训练产物会集中到 `out/`：

```text
out/pretrain_768.pth
out/pretrain_768_resume.pth
```

其中：

- `pretrain_768.pth` 适合做普通权重加载，即 `from_weight`
- `pretrain_768_resume.pth` 适合做断点续训，即 `from_resume`

一个常见的启动示例如下：

```bash
python .\trainer\train_pretrain.py --data_path E:/Downloads/pretrain_t2t_mini.jsonl --save_dir out
```

如果之后需要继续训练：

```bash
python .\trainer\train_pretrain.py --data_path E:/Downloads/pretrain_t2t_mini.jsonl --save_dir out --from_resume 1
```

如果只想基于已有纯权重继续启动：

```bash
python .\trainer\train_pretrain.py --data_path E:/Downloads/pretrain_t2t_mini.jsonl --save_dir out --from_weight pretrain
```

## 6. 小结
当前预训练路径相关代码可以概括为两句话：

- 数据集路径由 `train_pretrain.py` 传入，最终在 `dataset/lm_dataset.py` 中解析成可被 `load_dataset` 使用的绝对路径
- 模型保存路径由 `train_pretrain.py` 统一控制，纯权重、resume checkpoint、`from_weight`、`from_resume` 都围绕 `args.save_dir` 工作

这次修改的重点不是增加新能力，而是让当前训练入口的路径行为更一致、更容易理解。
