# MiniMindBlock
在 MiniMindBlock 里，可以把一层 Transformer Block 看成两段几乎一样的结构：
``` text
x -> Norm -> 子层(Attention/FFN) -> + x

即：
x = x + Attn(Norm(x))
x = x + FFN(Norm(x))    或      x = x + MOE(Norm(x))
```

代码组合为：
``` python
residual = x
x = self.norm1(x)
x = self.attn(x, ...)
x = x + residual

x = x + self.ffn(self.norm2(x))     # 或者 x = x + self.moe(self.norm2(x))
```

## 何为残差？
残差（residual）是指在神经网络中，输入与输出之间的差异。残差连接（residual connection）是一种特殊的连接方式，它将输入直接添加到输出上，以帮助缓解深层网络中的梯度消失问题。  
残差的本质：把模块的输出，和它的输入直接相加。
``` python
x = x + F(x)        # 其中 F(x) 可以是 Attention、MLP，或者别的子层
```

- 无残差连接：$x = F(x)$ 导致输出的信息中，原来的信息要全部“重新生成”，而不是“增量更新”。  
- 有了残差后：$x = x + F(x)$ x 作为“底稿”保留下来，F(x) 只负责补充、修正、增强，让网络从“重写”变成“改写”。

## 残差连接的作用
- 让信息不容易丢失  
  深层网络里，信息一层层往下传，容易被逐渐冲淡。残差给了输入一个直接通路，原始信息可以比较稳定地传下去。
- 让训练更稳定  
  反向传播时，梯度可以沿着残差这条“直路”传回去，不容易消失。
- 让每一层学“增量”  
  每层不需要把整个表示重做一遍，而是：
  - 哪些地方要加强
  - 哪些地方要修正
  - 哪些地方要抑制

## 为什么要先 Norm，再残差？
- 输入分布更稳定  
  每一层子模块看到的输入尺度比较稳定，不会因为前面层输出越来越大或者越来越飘，导致后面层难学。
- 深层训练更稳  
  Pre-Norm 通常比 Post-Norm 更容易训练深层 Transformer 模型。
- 残差通路更“干净”  
  残差分支保留的是原始信息，主分支负责“增量修正”。


## 为什么残差要加在子层输出之后？
子层本质上是在学习一个“修正量”，而不是“增量”。
``` python
x_out = x + F(Norm(x))
```
F(Norm(x)) 可以看成“对输入 x 的增量修改”：
- 如果某一层学不到有用东西，那它可以接近输出 0
- 那么整层就近似恒等映射 x_out ≈ x
- 模型不会因为某一层训练不好就把信息全毁掉


主路负责做复杂变换，旁路负责保留原信息：  
- 主路：Norm -> Attention/FFN  
- 旁路：Residual 直通  

主路负责做复杂变换，旁路负责保留原信息；没有残差，层数一深就很容易信息断掉；没有 Pre-Norm，训练会更不稳  



# MiniMindModel
该模块的目的：实现 transformer layers 模块的前向传播  

模块流程：
- 创建past_key_values储存空间
- 对输入的新文本，编码成token
- 对新的token进行相对位置编码
- 对新的token、past_key_values进行前向传播，N层Transformer Block
- 返回值进行dropout，得到最终的输出

# MiniMindForCausalLM
该模块的目的：实现语言模型的前向传播  

模块流程：
- 完成transformer layers模块的前向传播
- 获取指定长度 logits_to_keep 位置的 logits 
  - logits.shape = [batch_size, seq_len, vocab_size]，vocab_size中储存了分数
  - logits_to_keep = 0：返回所有位置的 logits：  
    训练（需要每个位置的预测来算 loss）  
    分析模型行为  
  - logits_to_keep = 1：返回最后一个位置的 logits：  
    生成（最常用）
  - logits_to_keep = n：返回长度n的位置的 logits：  
    想看最近几步预测变化  
    做对比分析  
    或某些自定义 decoding 策略  
- 计算训练损失
